# 🤖 Batalha de Modelos & Engenharia de Prompt (XML)

## 📝 Descrição do Projeto
Este projeto apresenta uma análise comparativa de desempenho entre diversos modelos de linguagem (LLMs) ao processar um prompt estruturado em XML. O objetivo foi avaliar a geração de uma página HTML Single Page com CSS integrado, seguindo diretrizes rígidas de design [1].

O experimento foi conduzido por Marcela Nascimento Silva, Raquel Ramos Nogueira e Matheus Cardoso [1], testando ferramentas como ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca [1]. O foco foi observar como a estrutura XML influencia a precisão do código e a eficiência no uso de tokens [1].

## 🚀 Tecnologias e Modelos Testados
* **Modelos:** Claude AI, Qwen, ChatGPT (GPT), Gemini, DeepSeek, Grok e Maritaca [1].
* **Linguagens Geradas:** HTML5, CSS3 e, em alguns modelos, JavaScript [2].
* **Metodologia:** Engenharia de Prompt baseada em tags XML para definição de objetivos e diretrizes [1].

## 📊 Resultados e Aprendizados
A análise revelou disparidades significativas entre as ferramentas:

* **Melhor Compreensão:** O **Claude AI** foi o modelo que melhor demonstrou entender a estrutura sugerida [2, 3].
* **Variação de Verbocidade:** O **Qwen** utilizou apenas 350 tokens, enquanto o **Claude** ultrapassou os 6.000 tokens para a mesma tarefa [3].
* **Qualidade Técnica:** O **DeepSeek** apresentou um prompt detalhista, gerando imagens com excelência e sem erros de sintaxe [2].
* **Limitações:** Modelos como o **Maritaca** e o **Grok** apresentaram dificuldades em atender às expectativas técnicas ou temáticas [1, 2].

## 🔧 Guia de Escolha
Com base nos testes [3]:
1. **Para Prototipagem Rápida:** **Qwen**, devido à agilidade e baixo consumo de tokens.
2. **Para Códigos Complexos:** **Claude**, por ser o mais criativo e expansivo.
